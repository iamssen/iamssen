# 2026-05-22
은퇴하고 딱히 하는 거 없이 취미 생활이나 하면서 지내던 중, Solid.js에 대해 알게 되었다.

예전에 RxJS + React로 Virtual DOM 렌더링을 과도하게 유발하지 않는 컨셉을 고민해 본 적이 있다. 하지만 어디서부터 어디까지 만들어야 할지 감이 오지 않을 정도로 큰 작업이 될 것 같아 관뒀었는데, Solid.js가 딱 그런 컨셉을 구현했더라.

더 이상 상업적 개발을 하지 않아 생산성을 위해 굳이 외부 라이브러리를 사용할 필요가 없어서 평소 UI 컴포넌트 라이브러리 없이 깡구현을 즐기는 편이다 보니 React 대신 Solid.js를 사용해 보는 것도 나쁘지 않을 것 같았다. (심심하기도 하고...)

Solid.js로 전환하면서 처음 맞닥뜨린 문제는 다음 두 가지다.

1. Solid.js 2.0 Beta 진행 중. API Breaking Change가 꽤 있음.
2. eslint-plugin-solid 관리가 안 되고 있음. 
  - ESLint v10 지원 안 됨 (메인테이너가 PR을 병합하지 않고 있는 상태)
  - Solid.js 2.0 규칙에 맞춰 추가로 구현할 게 있는지 확인해야 함

Solid.js 2.0이야 그냥 Beta로 설치해서 사용하면 되는데, eslint-plugin-solid는 좀 골치가 아프다. 일단 ESLint v10을 지원하는 PR을 사용해 직접 설치하는 방식으로 처리했는데, 2.0용 규칙들을 위해 추가적인 Rule을 구현해야 한다면 꽤나 머리가 아파질 것 같다.

뭐... 일단 해보자.
